cc# SEO, GEO & AEO Enhancement Module

> **Purpose:** This module is designed to be appended to an existing
> website or funnel prompt. It assumes the design system, UX,
> copywriting framework, and conversion strategy are already strong. Its
> purpose is to strengthen traditional SEO, Generative Engine
> Optimization (GEO), and AI Answer Engine Optimization (AEO) without
> changing the overall funnel architecture.

------------------------------------------------------------------------

# Core Instructions

The website already has a proven conversion-focused funnel and page
structure.

-   Do **not** redesign the funnel.
-   Do **not** recommend unnecessary pages.
-   Do **not** suggest a large blog, city-page network, or content hub
    unless there is a compelling business case.
-   Preserve the existing user experience unless a change measurably
    improves SEO, GEO, AEO, accessibility, or conversions.

Your objective is to optimize every existing page for discoverability,
authority, crawlability, semantic understanding, and AI citation
potential.

------------------------------------------------------------------------

# 1. Traditional SEO Optimization

Audit and improve every existing page for:

-   Search intent alignment
-   Primary keyword targeting
-   Secondary keyword coverage
-   Semantic keyword coverage
-   Topical completeness
-   Page titles
-   Meta descriptions
-   Canonical tags
-   Open Graph tags
-   Twitter/X Cards
-   Robots directives
-   XML sitemap recommendations
-   Internal linking
-   External authority signals
-   Heading hierarchy
-   Image optimization
-   Alt text
-   Lazy loading
-   Core Web Vitals
-   Crawlability
-   Indexability
-   Duplicate content risks
-   Thin content opportunities

Do not recommend keywords that do not match the business model or search
intent.

------------------------------------------------------------------------

# 2. GEO & AI Search Optimization

Optimize content so it is easily understood and cited by:

-   ChatGPT
-   Google AI Overviews
-   Perplexity
-   Gemini
-   Claude
-   Bing Copilot

Every major section should answer a clear user question.

Prefer:

-   Definition blocks
-   Short explanatory paragraphs
-   Comparison tables
-   Bullet lists
-   Step-by-step processes
-   Checklists
-   Summary boxes
-   FAQ sections
-   Glossary-style explanations
-   Entity-first writing
-   Semantic HTML

Avoid:

-   Keyword stuffing
-   Fluff
-   Generic marketing language
-   Unsupported claims
-   AI-sounding filler

------------------------------------------------------------------------

# 3. Entity Optimization

Identify every important entity on each page.

Consider:

-   People
-   Companies
-   Products
-   Services
-   Brands
-   Software
-   Industries
-   Concepts
-   Organizations
-   Locations

Explain how each entity should be reinforced through:

-   Copy
-   Headings
-   Schema
-   Internal links
-   Image alt text
-   Structured data
-   Consistent naming

Recommend missing entities that would strengthen topical authority.

------------------------------------------------------------------------

# 4. Schema Optimization

Recommend only schema that accurately represents the page.

Examples include:

-   Organization
-   Person
-   WebSite
-   WebPage
-   AboutPage
-   ContactPage
-   FAQPage
-   BreadcrumbList
-   VideoObject
-   ImageObject
-   Review
-   AggregateRating
-   Product
-   Offer
-   Course
-   Service
-   Article

Explain:

-   Why each schema is appropriate
-   Where it should be implemented
-   Which properties are required
-   How schemas should relate to one another

Do not recommend misleading schema.

------------------------------------------------------------------------

# 5. AI Citation Readiness

Improve content so AI systems can confidently summarize and reference
it.

Recommend:

-   Definition blocks
-   Key facts
-   Process summaries
-   Comparison tables
-   Original insights
-   FAQs
-   Consistent terminology
-   Clear section summaries
-   Retrieval-friendly formatting

Highlight any unsupported or ambiguous claims that reduce citation
confidence.

------------------------------------------------------------------------

# 6. E-E-A-T Optimization

Evaluate and strengthen:

-   Experience
-   Expertise
-   Authoritativeness
-   Trustworthiness

Identify missing:

-   Founder information
-   Author pages
-   Team information
-   Credentials
-   Testimonials
-   Reviews
-   Policies
-   Trust badges
-   Business transparency
-   Contact information
-   Citations
-   References

Recommend improvements that increase credibility with both users and
search engines.

------------------------------------------------------------------------

# 7. Competitive Gap Analysis

Compare the website against leading competitors.

Identify:

-   What competitors do well
-   What should be modeled
-   What should be avoided
-   Content gaps
-   Trust gaps
-   AI visibility gaps
-   Topical authority gaps
-   Technical SEO gaps

Recommend practical improvements without copying competitors.

------------------------------------------------------------------------

# 8. Technical SEO Review

Review:

-   Semantic HTML
-   Heading hierarchy
-   Internal links
-   URL structure
-   Robots directives
-   Sitemap
-   Canonicals
-   Performance
-   Accessibility
-   Mobile usability
-   Core Web Vitals
-   Structured data validation

Prioritize changes that improve both search visibility and user
experience.

------------------------------------------------------------------------

# 9. Deliverables

For every page provide:

-   Primary keyword
-   Supporting keywords
-   Search intent
-   Recommended title tag
-   Meta description
-   H1 recommendation
-   Schema recommendation
-   Internal linking opportunities
-   GEO improvements
-   AEO improvements
-   E-E-A-T improvements
-   Technical recommendations

------------------------------------------------------------------------

# Output Rules

-   Preserve the existing funnel structure whenever possible.
-   Recommend only meaningful improvements.
-   Separate facts from assumptions.
-   Flag unsupported claims.
-   Explain why every recommendation matters.

Tag every recommendation as one of the following:

-   **Critical**
-   **High**
-   **Medium**
-   **Low**

The final output should be immediately actionable for developers,
designers, SEO specialists, copywriters, and marketers.
